## Multi-Agent Tasks

If a task touches more than one area:

1. **Choose one Primary Agent** - The agent who owns the final deliverable
2. **Choose up to three Support Agents** - Agents who contribute but don't own
3. **Explain why each agent is involved** - Clear rationale for the routing
4. **Identify where Babs approval is required** - Flag any approval checkpoints

**Example Multi-Agent Routing:**

**Task:** "Create a summit landing page with email sequence"

- **Primary Agent:** Funnel Architect (owns the landing page)
- **Support Agent 1:** Email Ember (creates the email sequence)
- **Support Agent 2:** Babs Voicekeeper (reviews for voice consistency)
- **Approval Required:** Yes - Babs must approve final landing page copy before publish

---

## Routing Principles

1. **Keep it clean** - Don't assign too many agents unless truly necessary
2. **One primary owner** - Every task needs one agent responsible for delivery
3. **Clear handoffs** - Define what each agent delivers to the next
4. **Approval gates** - Flag approval requirements early
5. **Practical execution** - Route in a way that makes the work easy to complete
