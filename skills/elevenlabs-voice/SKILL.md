# ElevenLabs Voice

This skill provides comprehensive access to the ElevenLabs API, enabling advanced text-to-speech (TTS), voice cloning, and audio generation. It is designed for developers and content creators who need to integrate high-quality, dynamic voice synthesis into their workflows.

## Required credentials

- `ELEVENLABS_API_KEY` — Your ElevenLabs API key. You can obtain one from your [ElevenLabs Profile Settings](https://elevenlabs.io/app/settings/api-key).

## Helper script

For common operations, you can use the `scripts/elevenlabs_voice_api.py` helper script.

```bash
python scripts/elevenlabs_voice_api.py <command> [args]
```

| Command | Description |
|---|---|
| `tts --text TEXT --voice-id ID [--output PATH]` | Generates speech from text using a specified voice. |
| `list-voices` | Retrieves a list of all available voices in your library. |
| `get-voice --voice-id ID` | Fetches detailed information about a specific voice. |
| `add-voice --name NAME --file PATH` | Adds a new custom voice by providing a name and audio file. |
| `delete-voice --voice-id ID` | Deletes a custom voice from your library. |
| `generate-sound --text TEXT [--output PATH]` | Creates a sound effect based on a textual description. |
| `get-history` | Retrieves your audio generation history. |

## Workflow

### 1. Text-to-Speech (TTS)
**Endpoint:** `POST https://api.elevenlabs.io/v1/text-to-speech/{voice_id}`

```bash
curl -X POST https://api.elevenlabs.io/v1/text-to-speech/21m00Tcm4TlvDq8ikWAM \
  -H "xi-api-key: $ELEVENLABS_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Hello! I am a voice generated by ElevenLabs.",
    "model_id": "eleven_multilingual_v2",
    "voice_settings": {
      "stability": 0.5,
      "similarity_boost": 0.75
    }
  }' \
  --output speech.mp3
```

### 2. List Voices
**Endpoint:** `GET https://api.elevenlabs.io/v1/voices`

```bash
curl -X GET https://api.elevenlabs.io/v1/voices \
  -H "xi-api-key: $ELEVENLABS_API_KEY"
```

### 3. Get User Information
**Endpoint:** `GET https://api.elevenlabs.io/v1/user`

```bash
curl -X GET https://api.elevenlabs.io/v1/user \
  -H "xi-api-key: $ELEVENLABS_API_KEY"
```

### 4. Get History
**Endpoint:** `GET https://api.elevenlabs.io/v1/history`

```bash
curl -X GET https://api.elevenlabs.io/v1/history \
  -H "xi-api-key: $ELEVENLABS_API_KEY"
```

### 5. Get Available Models
**Endpoint:** `GET https://api.elevenlabs.io/v1/models`

```bash
curl -X GET https://api.elevenlabs.io/v1/models \
  -H "xi-api-key: $ELEVENLABS_API_KEY"
```

## Pagination

When retrieving lists of items such as voices or history, the ElevenLabs API uses cursor-based pagination. You can navigate through pages of results by using the `page_size` and `start_after_history_item_id` (for history) or similar query parameters. The response will contain a `last_sort_key` or `next_cursor` which you can use in the subsequent request to fetch the next page.

**Example: Paginate through history**
```bash
# Initial request
curl -X GET "https://api.elevenlabs.io/v1/history?page_size=5" \
  -H "xi-api-key: $ELEVENLABS_API_KEY"

# In the response, find the history_item_id of the last item, e.g., "xB2mC4oP4BvIqfP1x2a9"

# Subsequent request to get the next page
curl -X GET "https://api.elevenlabs.io/v1/history?page_size=5&start_after_history_item_id=xB2mC4oP4BvIqfP1x2a9" \
  -H "xi-api-key: $ELEVENLABS_API_KEY"
```

## Multi-step workflows

### 1. Create a Custom Voice and Generate Audio
This workflow demonstrates how to add a new voice to your library and then use it to generate speech.

1.  **Add a new voice**:
    ```bash
    python scripts/elevenlabs_voice_api.py add-voice --name "My Custom Voice" --file "path/to/audio.wav"
    ```
2.  **List voices** to get the new `voice_id`.
    ```bash
    python scripts/elevenlabs_voice_api.py list-voices
    ```
3.  **Generate audio** with the new voice.
    ```bash
    python scripts/elevenlabs_voice_api.py tts --text "This is my new custom voice." --voice-id <NEW_VOICE_ID> --output custom_audio.mp3
    ```

### 2. Generate Audio for a Blog Post
This workflow converts a text file containing a blog post into an audio file.

1.  **Read the content** of the blog post from a file.
2.  **Generate audio** using the text content.
    ```bash
    TEXT_CONTENT=$(cat blog_post.txt)
    curl -X POST https://api.elevenlabs.io/v1/text-to-speech/21m00Tcm4TlvDq8ikWAM \
      -H "xi-api-key: $ELEVENLABS_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "text": "'$TEXT_CONTENT'",
        "model_id": "eleven_multilingual_v2"
      }' \
      --output blog_audio.mp3
    ```

## Guardrails

- **API Key Security**: Never expose the `ELEVENLABS_API_KEY` in client-side code, logs, or public repositories.
- **Input Validation**: Always sanitize and validate user-provided text before sending it to the API to prevent injection attacks.
- **Voice Authorization**: Ensure that users are authorized to use or modify custom voices to prevent misuse.
- **Content Policy**: Adhere to ElevenLabs' content and usage policies. Do not generate content that is hateful, abusive, or illegal.
- **Error Handling**: Implement robust error handling to manage API failures gracefully.
- **Character Limits**: Be mindful of character limits for TTS requests, which vary by subscription tier.
- **File Size Limits**: When adding voices, respect the maximum file size for audio samples.
- **Data Privacy**: Be transparent with users about how their data (e.g., cloned voices) is being used and stored.

## Rate limits

The ElevenLabs API has rate limits that vary based on your subscription plan. Exceeding these limits will result in `429 Too Many Requests` errors.

- **Free Tier**: 30 requests per minute.
- **Creator Tier**: 60 requests per minute.
- **Pro Tier**: 120 requests per minute.

Always check the official [ElevenLabs documentation](https://elevenlabs.io/docs/api-reference/rate-limiting) for the most up-to-date information on rate limits.

## Failure handling

| Error Code | HTTP Status | Cause | Resolution |
|---|---|---|---|
| `invalid_api_key` | 401 | The provided API key is invalid or missing. | Verify that the `ELEVENLABS_API_KEY` is correct and has the necessary permissions. |
| `voice_not_found` | 404 | The requested voice ID does not exist. | Check the voice ID and ensure it is correct. Use the `list-voices` command to see available voices. |
| `validation_error` | 422 | The request body is invalid or missing required fields. | Review the API documentation for the correct request structure and parameters. |
| `rate_limit_exceeded` | 429 | Too many requests have been sent in a short period. | Implement exponential backoff and retry the request after a delay. Check your subscription's rate limits. |
| `server_error` | 500 | An unexpected error occurred on the ElevenLabs server. | This is a temporary issue. Wait a few moments and retry the request. If the problem persists, check the ElevenLabs status page. |
| `bad_request` | 400 | The request was malformed or could not be understood. | Check the syntax of your request, including headers and JSON payload. |
| `not_enough_characters` | 400 | Your character quota has been exceeded. | Check your usage and subscription details in your ElevenLabs account. Upgrade your plan if necessary. |
| `file_too_large` | 400 | The uploaded audio file for voice cloning is too large. | Reduce the file size of the audio sample or split it into smaller chunks if possible. |
